1-tap semantics

One widget tap writes a dose_logs row with taken_at = now(), dose_event = 'on_time', titration_step = 'steady'. Optional fields (site / context / dose override / notes) default to null and can be added later by opening the app.

Form-aware

When currentRegimen.form === 'pill' the widget copy swaps "dose" → "pill" and the syringe icon → pill icon. Everything else is identical.

Streak math

Streak count comes from consecutive on-time dose_logs. On successful log the post-tap state shows +1 to confirm the streak incremented. Resets on missed dose window.

Deep links

Medium and Large widgets expose secondary CTAs that open the app at specific routes — /log-weight, /log-side-effect. Small widget is tap-to-open-app for details; the primary tap stays on-widget.

Undo

Post-tap state persists ~6 hours. Tapping again within that window opens the log-dose screen pre-filled so the user can edit or delete. Prevents oops-taps without adding a confirm prompt.

Trust UI

No cohort language on the widget — just the user's own regimen and streak. "Illustrative demo data" disclaimers live inside the app. Widget stays as a personal log primitive.
