Trial Weave Flutter + Supabase

Production scaffold mirroring the React prototype. Same 17 screens, same Lokahi tokens, wired to Supabase auth + RLS-protected tables + a privacy-floored cohort RPC. This page lets you browse every file, the route map, and the schema.

Flutter 3.24+ Riverpod 2.5 go_router 14 supabase_flutter 2.5 fl_chart 0.68 20 files · 17 screens · 8 tables

What this delivers

A Flutter app that's screen-for-screen identical to the React prototype, with three real backend integrations the prototype only mocks:

  • Auth — email + Apple + Google via Supabase, with PKCE flow.
  • Per-user storage — RLS policies guarantee users only see their own rows.
  • Cohort RPC — server-side aggregate with a 20-person privacy floor; clients never see row-level data from other users.

What is NOT in scope

  • Push notifications (intentionally deferred — the spec calls for client-side reminders only at MVP).
  • HealthKit / Google Fit weight import — clean to add as a service in data/ later.
  • Apple/Google OAuth client config — code path exists, you'll need to set redirect URLs in Supabase + native side.

Tables & row-level security

Eight tables. Every table has RLS on with a single policy: auth.uid() = user_id. There is no path for one user to read another user's rows from the client SDK — cohort comparisons are computed exclusively through the cohort_outcomes RPC, which is SECURITY DEFINER and applies a 20-person privacy floor.

cohort_outcomes RPC

Server-side aggregation of weight-loss % by drug for users matching p_filters. SECURITY DEFINER + HAVING count(distinct user_id) >= 20 means the result set never reveals fewer than 20 people — drugs with a small cohort simply don't appear. Only authenticated roles can call it.

Get the app running

cd flutter_app
flutter pub get
flutter run \
  --dart-define=SUPABASE_URL=https://YOUR.supabase.co \
  --dart-define=SUPABASE_ANON_KEY=eyJ...

Provision Supabase

supabase db push   # applies supabase/migrations/0001_init.sql

The migration creates all 8 tables, enables RLS on each, installs the policies, and creates the cohort_outcomes RPC.

Native config (OAuth)

  • Apple — enable Sign in with Apple capability in Xcode; add the redirect URL in Supabase Auth → Providers.
  • Google — add a Google OAuth client (iOS + Android), configure URL schemes per supabase_flutter docs.
  • Deep links — register io.supabase.trialweave://login-callback/ in Info.plist + AndroidManifest.xml.
